# Seamless Auto-Update Implementation Plan

> **For agentic workers:** Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Enable Slack/Docker-style seamless auto-updates — background download, notification, one-click restart.

**Architecture:** Tauri's built-in updater plugin checks GitHub Releases for new versions. When found, downloads in background, shows toast notification, user clicks "Restart" to apply. No custom server needed.

**Tech Stack:** Tauri v2 plugin-updater, GitHub Releases API, Ed25519 signing

---

## Task 1: Generate Signing Keys

Tauri requires Ed25519 signatures to verify update integrity.

- [ ] **Step 1: Generate key pair**

```bash
cd kubilitics-desktop/src-tauri
npx @tauri-apps/cli signer generate -w ~/.tauri/kubilitics.key
```

This creates:
- `~/.tauri/kubilitics.key` (private key — NEVER commit)
- Outputs public key to stdout — save it

- [ ] **Step 2: Store private key as GitHub Actions secret**

Go to: https://github.com/kubilitics/kubilitics/settings/secrets/actions

Add secret: `TAURI_SIGNING_PRIVATE_KEY` = contents of `~/.tauri/kubilitics.key`
Add secret: `TAURI_SIGNING_PRIVATE_KEY_PASSWORD` = the password you set (or empty)

- [ ] **Step 3: Save public key in tauri.conf.json**

Will be done in Task 3.

## Task 4: Update Release Workflow to Sign and Generate Manifest

- [ ] **Step 1: Update release.yml**

File: `.github/workflows/release.yml`

In the Tauri build step, add environment variables:
```yaml
env:
  TAURI_SIGNING_PRIVATE_KEY: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY }}
  TAURI_SIGNING_PRIVATE_KEY_PASSWORD: ${{ secrets.TAURI_SIGNING_PRIVATE_KEY_PASSWORD }}
```

When these env vars are present, `tauri build` automatically:
- Signs all update bundles (.tar.gz on macOS/Linux, .nsis.zip on Windows)
- Generates `.sig` signature files alongside each bundle

- [ ] **Step 2: Add manifest generation step**

After the Tauri build step, add a step that creates `latest.json`:

```yaml
- name: Generate update manifest
  run: |
    VERSION="${{ github.ref_name }}"
    cat > latest.json << EOF
    {
      "version": "${VERSION#v}",
      "notes": "Kubilitics ${VERSION} release",
      "pub_date": "$(date -u +%Y-%m-%dT%H:%M:%SZ)",
      "platforms": {
        "darwin-aarch64": {
          "signature": "$(cat target/release/bundle/macos/Kubilitics.app.tar.gz.sig 2>/dev/null || echo '')",
          "url": "https://github.com/kubilitics/kubilitics/releases/download/${VERSION}/Kubilitics.app.tar.gz"
        },
        "darwin-x86_64": {
          "signature": "$(cat target/release/bundle/macos/Kubilitics.app.tar.gz.sig 2>/dev/null || echo '')",
          "url": "https://github.com/kubilitics/kubilitics/releases/download/${VERSION}/Kubilitics.app.tar.gz"
        },
        "linux-x86_64": {
          "signature": "$(cat target/release/bundle/appimage/Kubilitics.AppImage.tar.gz.sig 2>/dev/null || echo '')",
          "url": "https://github.com/kubilitics/kubilitics/releases/download/${VERSION}/Kubilitics.AppImage.tar.gz"
        },
        "windows-x86_64": {
          "signature": "$(cat target/release/bundle/nsis/Kubilitics_*_x64-setup.nsis.zip.sig 2>/dev/null || echo '')",
          "url": "https://github.com/kubilitics/kubilitics/releases/download/${VERSION}/Kubilitics_${VERSION#v}_x64-setup.nsis.zip"
        }
      }
    }
    EOF

- name: Upload update manifest
  uses: softprops/action-gh-release@v2
  with:
    files: latest.json
```

- [ ] **Step 3: Commit**

```bash
git add .github/workflows/release.yml
git commit -m "feat: sign desktop builds and generate update manifest for auto-updates"
```

## How It Works End-to-End

1. **App launches** → `UpdateChecker.tsx` waits 5 seconds, calls `check()` from `@tauri-apps/plugin-updater`
2. **Plugin checks** → fetches `https://github.com/kubilitics/kubilitics/releases/latest/download/latest.json`
3. **If newer version found** → shows toast: "Kubilitics v0.2.0 available"
4. **User clicks "Install"** → plugin downloads update in background, shows progress %
5. **Download complete** → toast: "Update ready — Restart now"
6. **User clicks "Restart"** → app restarts with new version applied

No manual download. No visiting GitHub. No drag-to-Applications. Just click and restart.
